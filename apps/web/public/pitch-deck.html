Workflow modes

Three ways to move a document through your team

Pick the mode that fits your situation — you can choose per document, not per account.

📁 Self-managed

You control distribution

Store and prepare the document in EasyDraftDocs. Download and distribute it yourself — email it, drop it in a shared drive, whatever you already use.

  • No routing, no external emails
  • Full audit trail still recorded internally
  • Good for: single-owner reviews, internal drafts
🏢 Internal only

Team signs inside the app

All participants are members of your workspace. No external emails sent — everyone signs directly in the platform.

  • Sequential or parallel routing
  • All signers have workspace accounts
  • Good for: internal approvals, policy acknowledgements
📬 Platform-managed

EasyDraft emails your signers

EasyDraftDocs emails each participant a secure, one-time signing link — in routing order. External signers need no account. They verify their identity via email code before signing.

  • External signers get a one-time link
  • Email verification code required before any signature action
  • Token consumed per external workflow send

Full audit trail

Every action — view, sign, request-changes, lock, reopen — is timestamped and stored. Version history tracks every document state.

🏁

Completion certificate

A downloadable certificate records every participant, timestamp, and the SHA-256 hash of the final rendered PDF.

🔏

Lock / reopen

Explicitly lock a document to prevent further changes. Reopen it if circumstances change — the full history is preserved.

🗃️

Private storage by default

Documents are stored in private Supabase buckets. Unsigned originals and signed outputs are kept in separate storage paths.

🗑️

Scheduled purge control

Documents are temporary-by-default. Opt in to retained storage, or let them purge on schedule — you set the policy.

🔄

Change-impact classification

Post-sign document edits are classified as non-material, review-required, or resign-required — so the record always reflects what everyone actually signed.

The signed record you can actually rely on

When a document completes, EasyDraftDocs renders all signatures into the final PDF and records a SHA-256 hash of that exact file. Every download generates the same hash — so you always know whether the file has changed.

Honest product summary

What EasyDraftDocs is — and what it isn't

We'd rather you know exactly what you're getting before you start your trial than be surprised afterward.

✓ What it does well

  • ✓ Upload any PDF and prepare it for signing without editing the layout
  • ✓ Place fields, assign participants, route sequentially or in parallel
  • ✓ Send external signers a one-time link — they don't need an account
  • ✓ Gate external signing actions behind an email verification code
  • ✓ Record a full audit trail and SHA-256 integrity hash on every document
  • ✓ Manage team access, billing, and shared tokens under one corporate account
  • ✓ Let signers place their own signature if no field was pre-assigned
  • ✓ Lock completed documents and reopen them if the workflow needs to resume

— What it doesn't do (yet)

  • — Full PDF content editing — EasyDraft is a workflow layer, not a layout editor
  • — Certificate-backed cryptographic signing (PAdES/CAdES) — the SHA-256 hash confirms file integrity, but embedded digital certificate signing is a planned next step, not live
  • — Legal compliance guarantees — check your jurisdiction's e-signature requirements before relying solely on EasyDraft for legally regulated documents
  • — Integrations or API access in the current plan — this is a standalone workflow tool
  • — Custom branding or white-labelling on the base plan
